# T-608: "About this project" wallboard tile + admin panel

**Agent:** gemini · **Milestone:** M6 · **Depends on:** T-004, T-404

## Context

Give the doorboard a self-describing "About this project" surface: what Doorboard
is, plus fun facts (lines of code, languages, and structural counts). The
wallboard is a public route, so only broad, build-time project facts are shown —
never private or diagnostic data.

Ambient tiles today are backed by static fixtures ([apps/door-ui/src/fixtures.ts](../../apps/door-ui/src/fixtures.ts)),
so this follows that pattern instead of adding a contract event (which would need
an ADR). The numbers are generated by a committed script and baked into a JSON
the UI imports; re-running the script refreshes them.

## Deliverables

- [tools/project-stats/collect.py](../../tools/project-stats/collect.py): counts
  lines of code by language across git-tracked files and tallies structural facts
  (services, packages, integrations, ADRs, task briefs, contract event types),
  writing `apps/door-ui/src/aboutStats.json`.
- `aboutFixture` in `fixtures.ts` wrapping the generated stats + a project blurb.
- A compact **"About Doorboard"** tile in the wallboard ambient grid (stat chips).
- A fuller **`AdminAboutPanel`** on `/admin` (description + language table + counts).
- door-ui vitest coverage; docs updated ([docs/ui/wallboard.md](../ui/wallboard.md), [docs/ui/admin.md](../ui/admin.md)).

## Out of scope

A live/worker-emitted stats event or new `packages/contracts` type (static baked
fixture by design). Historical trends. Restyling other tiles.

## Acceptance criteria

- Wallboard shows an About tile with LOC, language count, and structural counts;
  `/admin` shows the fuller panel with a per-language breakdown.
- Only public, build-time facts are displayed (no private/diagnostic data).
- `python tools/project-stats/collect.py` regenerates the stats deterministically.
- CI green (lint, typecheck, tests); no door-critical-path or contract changes.
